Build a full-stack golf tee time reservation system called "Pradera Verde Golf and Country Club" using Next.js (App Router), TypeScript, Tailwind CSS, and Convex as the backend. This is an MVP with mock payment only — no real payment gateway. Use Clerk for authentication.

---

## DESIGN SYSTEM — Apply to Every Page

### Color Tokens
- Primary green: #2D5016
- Accent gold: #C9A84C
- Background primary: #FFFFFF
- Background secondary: #F8F7F4
- Text primary: #1A1A18
- Text secondary: #6B6B67
- Border default: #E5E3DF
- Status — approved: #2D5016 (green)
- Status — pending: #B07D1A (amber)
- Status — rejected: #A32D2D (red)
- Status — cancelled: #6B6B67 (gray)
- Status — completed: #185FA5 (blue)

### Typography
- Display/headings: Georgia serif, normal weight, tight tracking
- Body: Inter or system-ui, 15–16px, line-height 1.65
- UI labels/eyebrows: 12–13px, uppercase, letter-spacing 0.08em, weight 500, color #6B6B67

### Component Rules
- Buttons: two variants only
  - Primary: bg #2D5016, white text, no border-radius above 6px, no shadow
  - Ghost: transparent bg, 1px solid #2D5016 border, #2D5016 text
- Inputs: 1px solid #E5E3DF border, 44px height, 14px label above (never inside/floating), 8px border-radius
- Cards: white bg, 1px solid #E5E3DF border, 12px border-radius, no drop shadow
- No icon-heavy card grids. No pill badge clusters. No decorative gradients or shadows.
- Section vertical padding: 80px desktop, 48px mobile
- All dividers: 1px solid #E5E3DF
- Loading states: simple text spinner or "Loading…" — no skeleton shimmer effects
- Error/success: inline message below the relevant field or action — no browser alert() calls

## CONVEX MUTATIONS AND QUERIES

Expose the following in Convex:

Queries:
- getAvailableSlots(date, courseId) → TeeTimeSlot[]
- getReservation(id) → Reservation + related user + slot
- getMyReservations(userId) → Reservation[]
- getAllReservations(filters) → Reservation[] (admin)
- getScheduleBlocks() → ScheduleBlock[]
- getNotifications(reservationId?) → Notification[]
- getCourse(id) → Course
- getDashboardMetrics() → { pendingToday, approvedThisWeek, activeSlots, upcomingBlocks }

Mutations:
- submitReservation(formData) → reservationId
- cancelReservation(id, userId) → void (golfer-initiated, checks ownership)
- updateReservationStatus(id, status, adminNotes?, customerFacingNote?) → void (admin only)
- confirmMockPayment(id, method) → void
- createTeeTimeSlot(data) → slotId
- updateTeeTimeSlot(id, updates) → void
- disableTeeTimeSlot(id) → void
- createScheduleBlock(data) → blockId
- deleteScheduleBlock(id) → void
- logNotification(reservationId, userId, event, channel) → void

All admin mutations must check ctx.auth and verify user role = "admin" before executing. All golfer mutations must verify user ownership where applicable.

## GLOBAL BEHAVIORS

- Convex real-time: reservation status changes on admin side must reflect immediately on golfer detail page (use useQuery live subscriptions)
- All form submissions show a loading state on the button ("Submitting…") until Convex confirms
- All Convex errors surface as inline error messages, never console-only
- After any admin status action, re-fetch reservation and log a notification record automatically
- Tee time slots with all spots taken (availableSpots = 0) must show as unavailable in the golfer booking grid even if status = available
- Schedule blocks must exclude matching date/time slots from the golfer-facing slot query result
